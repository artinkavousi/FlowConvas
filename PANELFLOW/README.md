# @artinos/panelflow

A self-contained, reusable, production-grade **panel system** for building interactive
visual tools and editor workspaces. Part of the **ARTINOS** creative studio ecosystem.

PANELFLOW provides everything you need to mount a premium, dockable editor workspace into
any React app: a panel runtime, an editor dock (bottom/left/right/float), a node graph
canvas, a frosted-glass design system, and — most importantly — a **Control Panel Engine**
that auto-generates control panels for any component from a simple parameter schema.

---

## Install

```bash
npm install @artinos/panelflow
```

```tsx
import { PanelFlowProvider, Workspace } from '@artinos/panelflow';
import '@artinos/panelflow/styles';
```

> `react` and `react-dom` (18 or 19) are peer dependencies. Three.js, R3F, etc. are
> demo-only and are **not** shipped with the package.

## Minimal integration

```tsx
import { PanelFlowProvider, EditorDock } from '@artinos/panelflow';
import '@artinos/panelflow/styles';

export default function App() {
  return (
    <PanelFlowProvider theme="dark">
      <div style={{ width: '100vw', height: '100vh', position: 'relative' }}>
        <MyViewport />        {/* your rendering content */}
        <EditorDock />        {/* the PANELFLOW workspace */}
      </div>
    </PanelFlowProvider>
  );
}
```

Or use the prebuilt `Workspace`, which wires the viewport slot, dock, and command palette:

```tsx
import { PanelFlowProvider, Workspace } from '@artinos/panelflow';

<PanelFlowProvider theme="dark">
  <Workspace viewport={<MyViewport />} />
</PanelFlowProvider>
```

## Auto-generated control panels

Register a component with a parameter schema and PANELFLOW generates a fully wired
control panel for it — no panel UI code required. The panel appears in the icon rail and
the command palette (`Ctrl+K`).

```tsx
import { registerComponent, useBridgeStore } from '@artinos/panelflow';

registerComponent({
  id: 'particle-system',
  name: 'Particle System',
  category: 'effects',
  parameters: [
    { key: 'count', label: 'Count', type: 'number', default: 1000, min: 100, max: 10000, step: 100 },
    { key: 'speed', label: 'Speed', type: 'number', default: 1, min: 0, max: 5, step: 0.1 },
    { key: 'color', label: 'Color', type: 'color', default: '#2dd4bf' },
    { key: 'gravity', label: 'Gravity', type: 'boolean', default: true },
    { key: 'position', label: 'Position', type: 'vec3', default: [0, 0, 0], min: -10, max: 10, step: 0.1 },
    { key: 'shape', label: 'Shape', type: 'enum', default: 'sphere', options: [
      { label: 'Sphere', value: 'sphere' },
      { label: 'Cone', value: 'cone' },
      { label: 'Box', value: 'box' },
    ]},
  ],
});
```

### Supported parameter types

| Type | Widget |
|------|--------|
| `number` (with `min`/`max`) | number input + gooey slider |
| `number` (no range) | number input |
| `color` | swatch + hex input |
| `boolean` | toggle |
| `string` | text input |
| `enum` | chip selector |
| `vec2` / `vec3` | per-axis number inputs + sliders |
| `range` | dual-thumb range |

### Reading values in your component

Parameter changes flow into the bridge store. Subscribe to apply them to your render:

```tsx
useBridgeStore.subscribe(
  (s) => s.componentValues['particle-system'],
  (values) => {
    mesh.material.color.set(values.color);
    mesh.scale.setScalar(values.count / 1000);
  },
);
```

Call `unregisterComponent(id)` to remove a component and its auto-generated panel.

## Public API

- **Provider:** `PanelFlowProvider`
- **Workspace / shell:** `Workspace`, `EditorDock`, `IconRail`, `PanelChrome`, `CommandPalette`, `ViewportSlot`
- **Panel OS:** `definePanel`, `usePanelOSStore`, `PANEL_REGISTRY`, `registerPanel`, `getPanelDefinition`, `PanelShell`
- **Control engine:** `registerComponent`, `unregisterComponent`, `getRegisteredComponents`, `generatePanelFromSchema`, `useBridgeStore`, `AutoGeneratedPanel`
- **Graph canvas:** `GraphCanvas`, `useGraphStore`, `UniversalNode`, `OSPanelNode`, `DotGridBackground`
- **Tweakpane Frost:** `useFrostPane`, `FrostPanePanel`
- **Design system:** `THEME`, `token`, `injectTheme`
- **Components:** `GooeySlider`, `BubbleRatingSlider`, `ElasticMenu`, `Tooltip`

Types: `PanelDefinition`, `PanelCapabilities`, `DockMode`, `ComponentSchema`, `ParameterDef`,
`ModifierDef`, `ControlBridge`, `FluidityNode`, `FluidityEdge`.
